# Contributing

Thank you for your interest in contributing to this academic research repository.

This project treats QoreChain as a blockchain research subject. Contributions should improve neutrality, technical accuracy, citation quality, structure, or long-term publication readiness.

## Contribution Principles

- Use academic and objective language.
- Avoid promotional or marketing claims.
- Distinguish clearly between official project documentation, independent analysis, and open research questions.
- Add citations or reference placeholders when introducing technical claims.
- Prefer structured analysis over broad assertions.
- Mark uncertain or future-facing claims as areas for further validation.

## Suitable Contributions

Examples of useful contributions include:

- Literature review additions
- Citation improvements
- Comparative analysis tables
- Technical architecture summaries
- Tokenomics methodology improvements
- Governance research notes
- Security and post-quantum cryptography references
- Formatting and repository organization improvements

## Writing Style

Use a neutral research tone. For example:

- Prefer: "QoreChain may be evaluated as a case study in post-quantum blockchain design."
- Avoid: "QoreChain is the best post-quantum blockchain."

Claims should be supported by references, official documentation, or clearly marked as hypotheses for future study.

## Pull Request Checklist

Before submitting a pull request, please check that:

- The contribution uses Markdown formatting consistently.
- New claims include citations or placeholders.
- Promotional language has been removed.
- The contribution aligns with the repository roadmap.
- References are listed or noted in `references/README.md`.
